Reflection
What I Learned: Through this assignment, I learned how to successfully simulate a real-world software development workflow using Git and GitHub
. I gained practical experience in initializing a local repository, tracking file changes through staging and committing, and linking my local repository to a remote server on GitHub
. I also learned how to isolate my work using branches and how to navigate the Pull Request workflow to merge new features into the main project
.
Challenges Faced: The main challenge I encountered was dealing with a merge conflict, as required in Task 6
. By intentionally modifying the exact same line in profile.txt on both the main and feature-update branches, Git could not automatically merge the files and halted the process
. It was initially intimidating to see the conflict markers (<<<<<<<, =======, >>>>>>>) injected directly into my text file. However, I learned how to carefully evaluate the conflicting lines, manually choose which changes to keep in my code editor, and then successfully finalize the merge commit.
How Git Helps in Real Projects (Improving Software Quality and Teamwork): Version control systems like Git drastically improve teamwork by allowing multiple developers to collaborate on the same project simultaneously. Branching enables developers to build and test new features in a safe, isolated environment without breaking the main codebase
. Furthermore, tools like GitHub enhance software quality through the Pull Request workflow
. Pull requests require team members to review, test, and discuss code before it is integrated
, catching bugs early. Because Git tracks a complete history of every change made, teams can also easily revert to a previous working version if a mistake does happen, ensuring a more stable and reliable software product.
